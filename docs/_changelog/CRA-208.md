# CRA-208 — C3: recurse — bounded self-referential Definition invocation

**What shipped.** `recurse(body, *, base_case, max_depth, combine, on_stuck, ...)` and the
`Recurse` node in `workflow.py`. Resolves the vision §5 open question: **recursion is a
depth-guarded `Program` back-edge re-entering the same FROZEN `Definition`, pushing a frozen
version onto a per-item depth stack.** It reuses the C2 kernel; the only deltas are a
**depth bound** (`max_depth`, distinct from a loop's `max_visits`) and a pure **base-case
predicate**.

**Mechanism.** Each descent level runs the frozen `body` once (the prior level feeds in as
a **FLUID** `_recurse_prior` input — taint propagates, never an instruction), then
`Output.derive`s a fresh content-addressed Output with `produced_by =
{body.content_sha()}#{edge_id}#d{depth}` (no in-place mutation — pushes the frozen version
onto the depth stack). The pure `base_case(output, depth)` predicate stops descent;
`combine` (an existing reducer — `collect`/`count`/`dedupe`) folds the descent-order
(depth-first) children into one Output. Halts on `base_case` / `depth >= max_depth` /
budget / cancel / calibrated no-progress — **never wall-clock**. `max_depth` is
**assembly-required**: a `None` bound raises **`UnboundedRecursionError`** at construction.

**Safety: depth is engine-authoritative (live-gate fix).** `base_case` receives
`(output, depth)`, where `depth` is the **trusted, engine-owned** 0-based index of the
level that just produced `output` — **not** a depth inferred from the (stochastic, possibly
marker-less) model Output. A body need not echo any depth/"am I done" marker, so a
termination decision read from fluid output is unsound (it can silently never fire and fold
0 parts — the live gate caught exactly this). The signature changed from
`Callable[[Output], bool]` to `Callable[[Output, int], bool]` so "how deep am I / am I done"
predicates run off trusted engine state. The first body call sees `depth == 0`; `base_case`
sees a contiguous `0, 1, 2, …` sequence
(`test_recurse.py::test_base_case_receives_authoritative_depth_sequence`, which uses a
constant marker-less body to prove the stop fires off engine depth, not Output content).

**Taint = union (security fix).** The `combine` taint rule is explicit: **the reduced Output
is tainted if ANY child input was tainted** (mirrors `run.py:261`). *"A vote/fold does not
launder taint."* (`test_recurse.py::test_combine_does_not_launder_taint`.)

**Determinism / durability.** Whole-tree shared `CostBudget`, preflighted each descent,
hard-kill on breach (`spent` reflects every level). Each level checkpoints into the F-2
**depth-variant** ledger (`checkpoint_depth(loop_id, item_id, depth, output_ref)`); resume
at depth *k* replays `1..k-1` at $0, content-hash-verified. `loop_id =
compute_loop_id(body.content_sha(), item_lineage, edge_id)` — deterministic, content-
addressed on the frozen body. Each level appends a distinct sha; rows carry `org_id`.

**Risk.** Tree fan-out is `O(b^d)`; the **shared budget is the real guard** (the cost
estimator OPT-2 folds `b^max_depth × per-iter`). `max_depth` + `base_case` are the
termination argument.

## Public API

- `recurse(body: Definition, *, base_case, max_depth, combine, on_stuck="return_last",
  progress=None, rubric_std=0.0, no_progress_patience=1, name="recurse") -> Recurse` —
  `base_case: Callable[[Output, int], bool]` (the `int` is the engine-authoritative depth).
- `Recurse.execute(seed, ctx, runtime, *, ledger=None, resume=False) -> RecurseResult`.
- `RecurseResult(output, depth_reached, stopped)`.
- `UnboundedRecursionError`.

**Decision (NodeKind).** `Recurse` is reduce-shaped (a tree of attempts → one folded
Output), so it reuses `NodeKind.AGGREGATOR` + a `node_kind_tag = "recurse"` for
telemetry/CLI — exactly as M1's `Refine` tagged itself — **without widening the core
`NodeKind` enum** (owned elsewhere).

**Acceptance test.** `test_recurse.py` — base-case fold-exactly-the-children, never-base-
case halts at `max_depth` (never exceeds `max_depth` calls), `UnboundedRecursionError` at
construction, budget hard-stop with per-level metering, taint-union, replay-identical fold
at $0, durable resume at $0, and `org_id` cross-org isolation.

**Ledger / hash implications.** Reuses the F-2 `ledger_loop` depth-variant
(`checkpoint_depth`/`completed_depths`) and `compute_loop_id` as-is — no schema change. The
recursion coordinate `{body.content_sha()}#{edge_id}#d{depth}` is a new derived
`produced_by` scheme; a change to it shifts every level's Output sha (a migration).
