# CRA-211 — AL-T4/TS-2: `cw.calibrate` — variance / calibration / abstention

`Benchmark.run` did one `Run` per task and meant the rubric, so run-to-run variance was
structurally invisible and the escalation threshold was a guessed constant. `cw.calibrate`
runs each golden case **N times under distinct, deterministically-derived seeds** and reports
the noise band, confidence calibration, and an evidence-derived abstention threshold — the
measurement layer the promotion gate (AL-T5) and the cost-regularized objective (AL-T3)
consume.

## Public API added (`crawfish.metrics`)

- `async calibrate(definition, golden, *, runs=5, ctx, runtime, rubric=None,
  confidence_field="confidence", cost_per_run_usd=0.0, target_accuracy=0.9, n_bins=10,
  alpha=0.05, n_resamples=1000, base_seed=0, inputs_for=None) -> CalibrationReport`
  - `golden` is a `GoldenSet` or an explicit `Sequence[EvalCase]`.
  - Drives the runtime directly via `RunRequest(decode_seed=...)` (not `Run.execute`, which
    does not plumb a per-call seed), so the per-run seed is the only stochastic point.
- `CalibrationReport(BaseModel, frozen)` — the `org_id`-tagged measurement. Fields the
  downstream gate / objective key off:
  - `rubric_mean: dict[str,float]`, `rubric_std: dict[str,float]` — per-metric mean and
    **population** std across the `runs × len(golden)` scored outputs (the noise band).
  - `output_variance: float` — mean fraction of structurally-differing fields across a case's
    re-runs (via `structural_diff`); `0.0` iff every re-run agreed (deterministic runtime).
  - `brier: float | None` — **primary** calibration metric (unbiased, no binning); `None`
    without labels.
  - `ece: float | None`, `ece_ci: tuple[float,float] | None` — ECE **diagnostic** (equal-mass
    / adaptive binning) and its **bootstrap CI**; both `None` without labels; `ece ∈ [0,1]`.
  - `reliability: tuple[ReliabilityBin,...]` — the equal-mass confidence→accuracy curve.
  - `abstention_threshold: float`, `abstention_rate: float` — the confidence below which acting
    is unsafe (read off `reliability`, not a constant) and the share of outputs below it.
  - `determinism_tier: DeterminismTier`, `infra_variance_floor: float` — the runtime's F-5 tier
    and, when it is not `honors-seed`, the variance floor attributed to **infra** (not the
    Definition).
  - `base_seed`, `runs`, `cases`, `content_sha`, `partial`.
  - `.gate_safe(margin) -> bool` — F-8 guard: forbids gating on `ece` when its CI is wider than
    the gate margin (and fails safe with no CI).
- `ReliabilityBin(BaseModel, frozen)` — `confidence`, `accuracy`, `count`.
- `CalibrationError(RuntimeError)` — raised on a `RecordReplayRuntime` (replay would zero
  variance — the one runtime calibrate legitimately drives non-replayed).

## Public API added (`crawfish.escalate`, new module)

- `extract_confidence(output, *, field="confidence") -> float | None` — read a `[0,1]`
  self-reported confidence from a typed Output (mapping field, or numeric value), clamped. The
  fluid value is **measured, never trusted as an instruction**.
- `abstention_threshold(bin_confidence, bin_accuracy, bin_count, *, target=0.9, default=1.0)
  -> float` — the evidence-derived replacement for the old guessed escalation constant: the
  lowest bin confidence above which observed accuracy stays `>= target`. Fails safe to `1.0`
  (abstain on everything) when no level is reliable or there is no evidence.

## Determinism & security (FORK resolutions)

- **Seed derivation** — per-run `decode_seed = random.Random(f"{base_seed}:{case_id}:{i}").
  getrandbits(63)` (the `FewShotMutator` discipline). Pure, reproducible, **static** (base seed
  recorded, case id is golden data, index positional — never fluid-derived). Same
  `(base_seed, runs)` over the same golden ⇒ identical seed schedule and identical report.
- **ECE definition** — equal-mass (adaptive) bins so a skewed confidence distribution stays
  populated; population-weighted `|confidence − accuracy|`; bootstrap CI re-bins each resample
  under a local seeded RNG. Brier is the primary (binning-free) metric.
- **Correctness is binary** — `_correct` is an exact structural-match indicator (not partial
  credit), so the reliability curve and Brier/ECE are not smeared.
- **Replay refusal** — `calibrate` raises `CalibrationError` on a `RecordReplayRuntime` rather
  than silently reporting `variance == 0`.
- **Autonomy ceiling** — each run charges `cost_per_run_usd` against `ctx.cost_budget` and
  checks `ctx.cancel_token` **before** the call; a breach returns a `partial=True` report over
  what was measured (the Tuner's ceiling-returns-base analogue), never unbounded spend.

## Acceptance (test_calibrate.py)

Seed-varied runtime ⇒ non-zero `rubric_std`/`output_variance`; deterministic runtime ⇒
`output_variance == 0` and zero std; same `(base_seed, runs)` ⇒ byte-identical report and seed
schedule; Brier computed, ECE `None` without labels else in `[0,1]` with a CI; a
perfectly-calibrated fixture (p=0.5, Bernoulli(0.5)) yields small ECE with a CI reaching ~0 and
Brier ≈ 0.25; report is frozen, carries `org_id`, records the determinism tier; replay refused;
budget/cancel return a partial report.

## Notes

`metrics.py` must not import `crawfish.eval` at module scope (`eval` imports `metrics`):
`EvalCase`/`GoldenSet` are `TYPE_CHECKING`-only; `RecordReplayRuntime` is imported lazily inside
`calibrate`. The report symbols (`CalibrationReport`, `calibrate`, `ReliabilityBin`,
`CalibrationError`) are exported from `crawfish.metrics.__all__`; the top-level
`crawfish.__init__` re-export is left to the package owner.
