# F-6 — Cost-model single owner + one composition law (CRA-199)

## What changed
`cost.py` is now the **single owner of the cost model**. The operator-aware cost
interval from OPT-2 is implemented here as a pure, deterministic layer on top of
the existing `estimate_cost` lower bound. No other module re-implements
estimation or re-defines an operator's cost multiplier.

- `CostEstimate` gains **additive** fields `expected_usd`, `worst_case_usd`,
  `expected_lo_usd`, `expected_hi_usd`. The existing scalar **`total_usd` is
  unchanged** — it remains the lower bound (every cost-bearing operator fires
  once). The new fields default to the lower bound, so every existing
  construction site and every existing test still passes (a bare estimate is a
  degenerate interval `[total, total, total]`).
- New `CostShape` describes one cost-bearing operator wrapper and its re-run
  multiplier, with ergonomic constructors that encode each operator's law once.
- New `compose_cost(base, shapes)` folds a nesting of shapes onto a base
  estimate, producing the expected/worst-case band.

## Public API added
```python
@dataclass(frozen=True)
class CostShape:
    kind: str
    worst_case_multiplier: float
    measured_rate: float | None = None      # measured fraction of calls that escalate/retry
    rate_ci: float = 0.0                     # half-width of the rate's CI (widens the band)
    strong_multiplier: float = 1.0           # escalation: strong_price / base_price

    def worst_case_factor(self) -> float: ...
    def expected_factor(self, *, ci_sign: float = 0.0) -> float: ...

    @classmethod
    def refine(cls, max_iters: int, *, measured_rate=None, rate_ci=0.0) -> CostShape: ...
    @classmethod
    def escalate(cls, *, base_price, strong_price, measured_rate=None, rate_ci=0.0) -> CostShape: ...
    @classmethod
    def quorum(cls, k: int) -> CostShape: ...
    @classmethod
    def retry(cls, n: int, *, measured_rate=None, rate_ci=0.0) -> CostShape: ...
    @classmethod
    def recurse(cls, *, branching: int, max_depth: int) -> CostShape: ...

def compose_cost(base: CostEstimate, shapes: Sequence[CostShape]) -> CostEstimate: ...

# CostEstimate new fields:
class CostEstimate(BaseModel):
    expected_usd: float
    worst_case_usd: float
    expected_lo_usd: float
    expected_hi_usd: float
```
Exported from `crawfish.cost.__all__` (`CostShape`, `compose_cost` added).
`__init__.py` was **not** edited (one-owner rule); import from `crawfish.cost`.

## The composition law (as implemented)
**Multiplicative along operator nesting.** `shapes` is the nesting
outermost-first; `compose_cost` folds the per-operator multipliers:

```
worst_case_usd = total_usd × Π shape.worst_case_factor()
expected_usd   = total_usd × Π shape.expected_factor()   (measured-rate band)
```

Per-operator worst-case factors:

| operator   | `kind`       | worst-case factor                         |
|------------|--------------|-------------------------------------------|
| `Refine`   | `"refine"`   | `max_iters`                               |
| `Escalate` | `"escalate"` | `1 + strong_price/base_price` (re-priced on the strong model) |
| `Quorum`   | `"quorum"`   | `k`                                       |
| `Retry`    | `"retry"`    | `n`                                       |
| `recurse`  | `"recurse"`  | `branching ** max_depth`                  |

Worked example (OPT-2): `Refine(4) ∘ Escalate(2×) ∘ Quorum(5)` over a base call
previews `4 × 2 × 5 = 40×` the lower bound.

**Escalation re-pricing.** The escalated attempt runs on the *strong* model, so
its marginal factor is `strong_price / base_price` (e.g. base $1, strong $5 ⇒
worst = base + strong = $6, not a flat 2×). A free/zero base price falls back to
the count-based 2×.

**Expected band (CI-aware, never falsely precise).**
`expected_factor = 1 + p·(worst_case_factor − 1)` where `p` is the operator's
`measured_rate` (from `cw.calibrate`/ledger). `rate_ci` shifts `p` by ±ci to
build the `expected_lo`/`expected_hi` edges, so `expected` is always a band.

**Never undercount.** With **no** measured rate, `expected_factor ==
worst_case_factor`, so `expected_usd == worst_case_usd`. A measured rate of 0
collapses `expected` to the lower bound (extra work never fires). The model
validator enforces the invariant
`total_usd ≤ expected_lo ≤ expected ≤ expected_hi ≤ worst_case` and rejects any
malformed interval at construction.

**Purity / determinism.** `compose_cost` and every `CostShape` method are pure
functions of their inputs — no model call, no ledger read, no mutation. The base
estimate flows through the *same resolver the runtime uses* via `estimate_cost`.

## Single owner — CL-3 and ALG-5 are consumers, not editors
`cost.py` owns the cost model. Downstream issues **call** this API and must not
re-implement it:
- **CL-3** (cost-aware `Refine` preflight / "next-call-fits") reads
  `worst_case_usd` / `expected_usd` — it does not re-derive operator multipliers.
- **ALG-5 / AL-T3** (cost-regularized `Objective`) takes the per-candidate
  `cost_usd` from a `CostEstimate` here.
- The `craw dev --estimate` surface prints the three-number interval
  (`total` / `expected` / `worst_case`); `--json` includes all three plus the
  `expected_lo`/`expected_hi` band.
