# F-5 — Decode-knob ownership (CRA-198)

Settles who owns each decode parameter, in code and in ADR 0017.

## Public API added

**`crawfish.definition.types`**
- `AgentSpec.temperature: float | None = None` — tunable decode knob (in the content hash).
- `AgentSpec.top_p: float | None = None` — tunable decode knob (in the content hash).
- `AgentSpec.sample_k: int | None = None` — tunable top-k cutoff (in the content hash).
- `AgentSpec.decode_knobs() -> dict[str, float | int]` — the non-None knobs.
- `Definition.content_dict() -> dict[str, object]` — canonical hash payload; drops None decode
  knobs and the volatile `version`/`id`. The migration seam (see below).
- `Definition.content_sha() -> str` — canonical 12-char content hash over `content_dict()`.
- `Definition.resolved_decode(role=None) -> dict[str, float | int]` — the authoritative decode
  config for a role (lead → first agent fallback). The one run-time read path.
- `CONTENT_HASH_VERSION = 1` — bumped because the hashed field-set changed.
- `DECODE_KNOB_FIELDS = ("temperature", "top_p", "sample_k")`.

**`crawfish.runtime.base`**
- `DeterminismTier((str, Enum))` — `HONORS_SEED` / `BEST_EFFORT` / `NONE`.
- `RunRequest.grammar: str | None = None` — per-call constrained-decode grammar (NOT hashed).
- `RunRequest.decode_seed: int | None = None` — per-call seed; F-1 folds it into the cassette
  key (NOT hashed).
- `RunRequest.resolved_decode() -> dict[str, float | int]` — derives the turn's knobs from the
  Definition.
- `RunRequest.temperature` — **read-only property**, derived from the resolved Definition. There
  is no settable `temperature` field (one authoritative location; the request value is derived).
- `AgentRuntime.determinism_tier: DeterminismTier = BEST_EFFORT` — backends advertise their tier.

## Migration / content-hash bump

- `CONTENT_HASH_VERSION` introduced at **1** to mark the addition of decode knobs to the hashed
  field-set.
- **Hash-neutral when None:** `Definition.content_dict()` drops any decode knob that is None.
  An existing artifact that never pinned a knob hashes **byte-identically** to its pre-v1
  payload → its `version.sha` does **not** change → **no re-freeze required**.
- An artifact that **pins** any knob (`temperature`/`top_p`/`sample_k`) gets a **new sha** and
  must be re-frozen — correct, since pinning a decode knob makes it a different artifact.
- Verified: no existing test pins a literal sha; full suite green (736 passed, 1 skipped).
