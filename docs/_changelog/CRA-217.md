# CRA-217 — TS-7/R4: House-guard (learned-then-distilled deterministic guards)

The deepest expression of the thesis: a program *accretes its own deterministic
invariants*. Quality is **learned stochastically**, **distilled** to a pure predicate,
and only **earns** enforcement after clearing an absolute precision-and-coverage bar.

## What shipped

New module `crawfish/guard.py`:

- **The pipeline.** `from_corrections` (F-4, reused) mines a `GoldenSet` → `propose_rule`
  (the ONE stochastic leaf — a single model `Run`) emits a FLUID candidate → `distill`
  parses it *as data* into a closed predicate grammar → `synthesize_guard` validates it
  against the corpus on a **joint** criterion and earns (or withholds) enforcement.
- **Closed predicate grammar** (`Predicate` = `Comparison | SetMembership | NumericBound
  | BoolCombination | Always`): a fixed, total, side-effect-free expression AST over typed
  Output fields, evaluated by an interpreter that **never** uses `eval`/`exec`. The FLUID
  proposal can only *select within* the grammar — an unknown kind/operator/term raises
  `GuardGrammarError` (the proposal cannot *widen* the grammar). The distilled AST becomes
  STATIC only after the precision gate.
- **Joint earn-the-right-to-block gate** (`GuardCertificate`): reports **both** a precision
  **lower bound** (Wilson) **and** coverage with a CI. Graduation is `precision_lb >=
  precision_floor AND coverage.lo >= min_coverage AND corpus non-empty`. A
  99%-precision / 2%-coverage rule therefore **cannot** block. **Fails closed**: no corpus
  ⇒ stays in `warn`; `HouseGuard.require_earned()` raises `GuardNotEarned`.
- **`HouseGuard`**: holds the distilled `Predicate` + certificate + `GuardStage`
  (shadow→warn→block). `blocks()` enforces only in `BLOCK`; `matches()`/`as_metric()` are
  pure observation. Content-hashed (`content_sha` over the predicate AST — a new rule mints
  a new sha, never an edit), carries `org_id`, reversible.
- **`PredicateMetric`**: the distilled predicate exposed as a pure `Metric` (0/1, $0).
- **Stats** (`wilson_lower_bound`, `proportion_ci`): stdlib-only, deterministic, built on
  the F-8 `experiment.normal_ppf` substrate.

## Precedents mirrored

- The `Verifier` *earn-the-right-to-gate* discipline + shadow→warn→block lifecycle.
- The F-3 absolute-precision gate's fail-closed posture, widened to the joint criterion.
- The F-4 `from_corrections` provenance/taint gate (Gap S4) is the corpus-poisoning
  control — a fluid-derived correction can never become a guard's ground truth.

## Security

The guard is a **consequential authority** (it blocks outputs). Fluid never *sets* the
guard: the proposer emission is FLUID and is parsed as data into the closed grammar;
authority is conferred only after the corpus (trusted, untainted corrections) clears the
joint bar. Taint propagates from any fluid input into the certificate (`tainted`).

## Determinism

Only `propose_rule` calls the model (replays via cassette). The distilled predicate is
pure (same input ⇒ same 0/1, $0); validation is pure arithmetic; each synthesis mints a
content sha. Full test suite runs with no live model call.
