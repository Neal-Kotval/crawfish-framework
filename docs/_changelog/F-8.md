## What changed
Added the **shared statistical substrate** for the gate algebra and the
normative spec that governs it. Two deliverables:

- `docs/architecture/experiment-design.md` â€” the cross-cutting **acceptance
  gate**. Every statistical consumer (`calibrate` / `gate` (F-3) / `quorum` /
  `explore` / `guard`) must cite it. No gate ships until it conforms. Covers
  estimand definitions, primary-vs-guardrail designation, pre-registered `n` vs
  anytime-valid sequential bounds (peeking control), paired tests over the
  GoldenSet, Holm family-wise correction, the held-out tune/gate split, power /
  MDE sizing, and winner's-curse correction.
- `crawfish.experiment` â€” a **pure, deterministic, stdlib-only** module of the
  adoption hooks F-3 and later milestones import. No model calls; any bootstrap
  is seeded explicitly via a local `random.Random` (never the global RNG), so
  identical inputs+seed yield byte-for-byte identical output across processes.

This module does **not** edit `metrics.py` or `eval.py`. F-3 will import these
primitives into the gate; F-8 only provides them.

## Dependency choice
The repo ships **no numpy/scipy** (`pyproject.toml` deps are `pydantic`,
`typing-extensions`, `pyyaml`). All primitives use the **standard library only**
(`math`, `statistics`, seeded `random.Random`). `normal_ppf` is a pinned Acklam
rational approximation so the `k`-from-`Î±` mapping is reproducible across
interpreter versions rather than depending on `statistics.NormalDist.inv_cdf`.

## How F-3 consumes it
- The variance-aware gate (b) uses `paired_bootstrap_ci` on per-case deltas:
  CI strictly above 0 â†’ adopt; CI straddling 0 â†’ reject (noise band).
- `k_from_alpha` replaces the magic noise-band constant; at the `k=0, std=0`
  limit the gate reproduces today's `is_regression`/`gate_against_baseline`
  byte-for-byte.
- The Tuner gates on `tune_gate_split`'s held-out `gate` set, never the `tune`
  set it searched.
- A promoted argmax is passed through `winners_curse_shrink` before
  `save_baseline`.
