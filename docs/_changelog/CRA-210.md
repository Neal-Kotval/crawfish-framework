# CRA-210 — AL-T2: `state_dict()` / `load_state()` (references-by-version transfer)

**Milestone 3 (Tunable ML library). R5 — "Hugging-Face-for-agent-weights".**

The architecture/weights split. A `Tuner`/`LearningLoop` winner was an opaque
whole-`Definition` blob — you could not transfer what it *learned* onto a sibling
Definition of the same shape, share it across a fleet, or A/B two knob settings on one
architecture. `state_dict()` is the split.

## What shipped (`packages/crawfish/src/crawfish/learning.py`)

- **`StateDict`** (Pydantic, frozen, JSON-only) — the tunable knobs as the "weights":
  - `roles: dict[str, RoleKnobs]` — per-role `prompt` / `model` / `context_strategy` /
    `policies` / decode knobs (`temperature`/`top_p`/`sample_k`, carried only when pinned).
  - `coordination: Coordination` — the team topology *choice* (a tunable knob).
  - `injected_prompts: list[Prompt]` — few-shots / appended instruction blocks.
  - `summons: list[DefinitionRef]` — summoned units as **references-by-version**
    (`{id, version}`), never an embedded nested Definition (vision §5 open Q). Embedding a
    full Definition is rejected at validation (the field type is `DefinitionRef`).
  - `structure_sha` — content hash of the **architecture** (sorted role set, typed IO
    schema, dependency ids, coordination kind) — the transfer-compatibility key.
  - `sha` (property) — content hash of the knob **values**; editing any knob changes it.
- **`RoleKnobs`** — the per-role knob bundle (frozen Pydantic).
- **`state_dict(definition) -> StateDict`** — extracts the knobs; **excludes** architecture
  keys (no `inputs`/`outputs`/`dependencies`/`team` topology in the payload).
- **`load_state(definition, state, *, strict=True, only=None) -> Definition`** —
  **copy-on-write** via the Tuner's `_refreeze` (a NEW frozen Definition, fresh content
  sha; the target is never mutated in place).
  - `strict=True` raises **`IncompatibleStateError`** on a `structure_sha` mismatch;
    `strict=False` loads the structural **intersection** (shared roles only).
  - `only=[...]` restricts the transferred knob groups — members of
    `{prompt, model, context_strategy, policies, decode, fewshots, coordination}`
    (e.g. `only=["fewshots"]` transfers only the injected few-shot prompts).

## Acceptance pinned (`packages/crawfish/tests/test_state_dict.py`)

- `state_dict()` excludes architecture keys; editing a knob changes `StateDict.sha`; a knob
  edit leaves `structure_sha` unchanged (still transfer-compatible).
- `d.load_state(d.state_dict())` is **sha-identity** (re-mints the same content sha).
- `strict=True` raises on a shape mismatch; `strict=False` loads the intersection.
- `only=["fewshots"]` / `only=["model"]` transfer only that group.
- Summons are `{id, version}` refs; embedding a nested Definition is rejected.
- `load_state` is copy-on-write (target unchanged, new frozen artifact).

## Security spine

Only STATIC knobs cross — every transferred field (prompt/model/policies/decode/
coordination/few-shots) is author/static config, never a fluid value. A consequential knob
(model/policies) that transfers is still static config, never a fluid-derived Sink target.
JSON-only ⇒ loading a state never executes code. References-by-version keep the hash
bounded and replay reproducible.

**Gap S4 (carried, not closed here):** few-shots may carry data mined from fluid/customer
corpora via `capture_case`/`from_corrections`. Cross-org `load_state` (the fleet-sharing
vision) still needs a data-governance/PII review and taint/tenancy tracking on shared state
dicts — scrubbing covers secrets, not embedded training data.
