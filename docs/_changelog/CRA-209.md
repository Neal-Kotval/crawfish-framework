# CRA-209 — AL-T1: two-axis mode unifier (per-knob `tunable` + `train()`/`eval()`)

PyTorch's hardest lesson — `requires_grad` and `.eval()` are *orthogonal* axes — applied to
Crawfish. Which knobs may move (Axis 1, tunable) and whether the artifact is sealed (Axis 2,
mode) are now distinct, and unified in `crawfish.tuner`.

## Public API added (`crawfish.tuner`)

**Axis 1 — `tunable` as content-hashable data**
- `KnobDomain(BaseModel, frozen)` — one knob: `path: str`, `values: list[KnobValue]`,
  `tunable: bool = True`. `path` is a dotted address into the existing knob vocabulary
  (`agent.<role>.model` / `.prompt` / `.temperature` / `.sample_k` / `.context_strategy` /
  `.policies`, `team.coordination`, `injected_prompts`).
- `TuneSpec(BaseModel, frozen)` — `knobs: list[KnobDomain]`. The typed form of `tune.toml`.
  - `named_knobs() -> Iterator[tuple[str, KnobDomain]]` — yields only `tunable=True` knobs,
    **path-sorted** (set/insertion-order-free).
  - `is_tunable(path) -> bool` — declared-and-tunable test (unknown paths are not tunable).
  - `from_toml(text) -> TuneSpec` — parses the `[[knob]]` array-of-tables authoring shape.
  - `to_dict() -> dict` — canonical, path-sorted, JSON-ready payload for export + hashing.
- `tune_spec_sha(spec) -> str` — deterministic 12-char content hash of a `TuneSpec`. Empty
  spec hashes to a stable constant (an empty `tune.toml` is hash-neutral).

**Axis 2 — `train()` / `eval()` mode**
- `train(defn) -> Definition` — **train mode**: an *unfrozen* deep copy with a fresh
  `Version` (`frozen is False`, `version.sha is None`). Copy-on-write: a training mutation
  mints a new `version.sha` when re-frozen, never edits the original in place.
- `eval(defn) -> Definition` — **eval mode** (the default): re-freezes to the canonical
  `Definition.content_sha()`. `eval(train(d))` re-hashes to `d`'s eval sha (idempotent);
  diverges only when a knob is actually edited.
- `guard_consequential(defn) -> None` — the load-bearing gate. Raises `FrozenError` unless
  `defn` is eval-mode (frozen); a no-op in eval mode. The single call every consequential
  boundary (a Sink write, a recorded run) makes before committing an irreversible effect.

## Behavior / invariants

- **Sink-in-eval-only.** A consequential side effect against an unfrozen (train-mode)
  Definition raises via `guard_consequential`; against an eval-mode Definition it succeeds.
  (A training artifact has no stable content identity to key idempotency / attribute the
  effect, so the effect is forbidden.)
- **TuneSpec-driven mutation refuses a pinned knob.** `named_knobs()` proposes exactly the
  tunable set a hand-built mutator would, and never offers a `tunable=False` knob.
- **Determinism.** `train()`/`eval()` only ever mint a frozen sha via the content-hash path;
  `TuneSpec` is static config in the hash; mode adds zero stochasticity.

## Security spine

`KnobValue` is a narrow scalar leaf: a knob *domain* is static, author-supplied config (it
enters the content hash), never a fluid / model-derived value. `train→eval` is a
content-hashed freeze. The consequential boundary is eval-only.

## Internal change

- `crawfish.tuner._content_sha` now delegates to the canonical `Definition.content_sha()`
  (ADR 0017 / F-5) instead of re-implementing a hash — it drops `version` **and** `id` and
  folds in the decode knobs (hash-neutral when None). One source of hash truth.
