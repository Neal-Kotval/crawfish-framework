# F-1 — Canonical cassette-key / execution-coordinate schema (CRA-194)

Single owner: `packages/crawfish/src/crawfish/runtime/replay.py`.

## What changed

`_key(request)` — the cassette identity / **execution coordinate** of a run — now folds
three new components beyond the legacy core, each **only when non-default** so the legacy
key is reproduced byte-for-byte when none are set.

### New `_key` component order

Canonical dict (serialized with `json.dumps(..., sort_keys=True)`, so insertion order is
cosmetic; the *set* of keys is what matters):

1. `id`          — `definition.id` (legacy core)
2. `version`     — `str(definition.version)` (legacy core)
3. `role`        — resolved agent role (legacy core)
4. `model`       — per-run model override (legacy core)
5. `inputs`      — bound run inputs (legacy core)
6. `session_id`  — resume token (legacy core)
7. `coordinate`  — **NEW**, appended only when a non-empty `ExecutionCoordinate` is passed
8. `org_id`      — **NEW**, appended only when `!= "local"`
9. `decode_seed` — **NEW**, appended only when present and non-`None` on the request

`_key` signature is now:

```python
def _key(request, *, org_id="local", coordinate=None) -> str
```

`request` stays positional, so existing callers — notably `cache.py`'s `cache_key`
(`_key(request)`) — are unchanged and keep producing the legacy key.

### `ExecutionCoordinate` API

A frozen dataclass; every axis is `Optional[int]`, default `None`:

```python
@dataclasses.dataclass(frozen=True)
class ExecutionCoordinate:
    sample_index: int | None = None   # quorum / best-of-k sample (0..k-1)
    iter_index:   int | None = None   # Refine iteration
    visit_count:  int | None = None   # MCTS / search visit
    depth:        int | None = None   # recurse recursion depth

    def is_empty(self) -> bool: ...       # all axes None ⇒ folds nothing
    def as_canonical(self) -> dict: ...    # non-None axes only, for content hashing
```

An all-`None` coordinate (`ExecutionCoordinate()`) is treated as **absent** and folds
nothing — preserving legacy keys. Exported from `crawfish.runtime.replay`
(`__all__` now includes `ExecutionCoordinate`).

### Runtime plumbing

`RecordReplayRuntime.run(request, ctx, *, coordinate=None)` gained an optional
keyword-only `coordinate`. It reads `org_id` from `ctx.org_id` and threads both into
`_key`. Existing callers that pass nothing get identical behaviour. Operators that re-run
a leaf (quorum, Refine, MCTS, recurse) stamp their axis so each re-run gets a distinct
cassette instead of colliding into one.

## Back-compat guarantee (pinned)

With **no** coordinate, `org_id == "local"`, and **no** decode field, `_key` produces the
exact pre-F-1 key. Pinned in `test_replay_key.py`:

```
LEGACY_KEY = "8dd9f4eb30b6b0ed"
```

(for `RunRequest(role="scout", inputs={"pr_body": "x"})` over the `full` fixture). The
full suite (660 passed, 1 skipped) — including all pre-existing replay/cache tests —
stays green, which is the live proof legacy unsalted cassettes still resolve.

## Flags for sibling docs

- **ARCHITECTURE.md (seam/contract change):** `_key` is the cassette / execution-coordinate
  contract. Its component schema is now versioned and extensible; record this as the
  canonical run-identity definition that every leaf-re-running operator must stamp.
- **SECURITY.md (tenancy):** `org_id` now enters cassette identity — cross-tenant cassettes
  can no longer collide. Decode-control fields (`decode_seed`) that are *not* in the
  Definition content hash now enter run identity, so two decode settings can no longer
  replay identically (closes the TS-8 hole). `decode_seed` is read defensively via
  `getattr` (F-5 may add the field to `RunRequest` concurrently); `base.py` was **not**
  edited by this issue.

## Migration / ADR needed

This does **not** change any existing key (legacy is byte-for-byte preserved), so no
migration is forced today. But folding `org_id` / `coordinate` / `decode_seed` is a
forward-compatible **hash-schema extension**: an ADR should record the schema, the
"fold-only-when-non-default" back-compat rule, and a **hash-bump / migration policy** for
any future change that *would* move legacy keys (e.g. promoting a now-optional component
to always-on, or adding a new always-folded field). Until that ADR lands, no component may
be made unconditional.
