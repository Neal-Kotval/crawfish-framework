# F-3 — The gate algebra (single owner of the eval.py gate) (CRA-196)

## What changed
One owner now reconciles the **three gate notions** that were conflated across
the drafts, and names which consumer uses which. All three consume the shared
statistical substrate `crawfish.experiment` (F-8 / CRA-201) — **no stats are
re-implemented** in `eval.py`/`metrics.py`. The normative spec is
[`docs/architecture/experiment-design.md`](../architecture/experiment-design.md).

### The three gates and their consumers

| Gate | Function | Who uses it |
|------|----------|-------------|
| (a) relative-regression | `metrics.is_regression` / `eval.gate_against_baseline` (**unchanged**) | Cheap aggregate-score callers that retain only mean scores; today's zero-tolerance behaviour. |
| (a′) variance-aware aggregate sibling | `metrics.is_regression_variance_aware` (**new**) | Callers that retain a recorded per-metric `std` but not per-case deltas. At `std=0` reduces to (a) byte-for-byte. |
| (b) variance-aware **paired** | `eval.paired_gate` (**new**) | The Tuner / `calibrate` / promotion gate (AL-T5/TS-2). Baseline + candidate scored on the **same** GoldenSet cases ⇒ paired per-case deltas. |
| (c) absolute-precision | `eval.precision_gate` (**new**) | Verifiers / guards / consequential sinks (CL-2/TS-3/R4). **Fails closed.** |

## Adopted statistical corrections (hard requirements from F-8)
- **Paired test over per-case deltas.** `paired_gate` analyses
  `candidate_i − baseline_i` over identical GoldenSet cases via
  `experiment.paired_bootstrap_ci`. CI strictly above 0 (`lo > 0`) ⇒ promote; a
  CI straddling 0 (within the noise band) ⇒ reject. Never an unpaired normal band.
- **Family-wise correction OR primary+guardrails.** `paired_gate` selects the
  design via its `guardrails` argument:
  - `guardrails is None` ⇒ Holm family-wise correction (`experiment.holm_correction`)
    across the rubric's metrics, so a rich rubric cannot inflate false-promotion
    past `α`.
  - `guardrails` given ⇒ one primary improvement target + pre-registered
    non-inferiority guardrails (each guardrail's lower CI bound must clear
    `−margin`).
- **`k` derived from `α`.** The noise-band multiplier `k` comes from
  `experiment.k_from_alpha` (`metrics.noise_band`), never a magic constant.
- **Clopper-Pearson dropped for continuous `[0,1]` rubrics** (per spec §1); the
  paired bootstrap is the canonical interval. Clopper-Pearson is reserved for the
  genuinely binary precision gate's underlying counts only.

## The fail-closed precision gate (CL-2 safety inversion — flag SECURITY.md)
`eval.precision_gate(decisions, labels, *, min_precision, baseline_exists)` is an
**absolute** decision-quality gate, not a relative one. Precision is
`TP / (TP + FP)` against the labelled decision GoldenSet. It **fails closed**:

- `baseline_exists is False` (never benchmarked) ⇒ **raises `VerifierNotGated`**
  — an un-measured verifier is rejected by construction, never admitted by default.
- No positive predictions (precision undefined) ⇒ raises (fails closed).
- Measured precision `< min_precision` ⇒ raises.

This is the **CL-2 safety inversion fix**: the old default was "admit unless
proven bad"; the new default is "reject unless measured good." **SECURITY.md
should record this**: consequential verifiers/guards (`VerifierStop`,
consequential sinks) must pass `precision_gate` with a real baseline before they
gate anything.

## Back-compat: `std=0, k=0` reproduces `is_regression` BYTE-FOR-BYTE
`is_regression_variance_aware` widens each metric's tolerance by that metric's
`k * std` band. When every `std` is `0` (or `std is None`) the band is
zero-width, so the comparison `delta < -(tolerance + k*std)` collapses to
`delta < -tolerance` — **identical** to `is_regression`. Pinned by
`test_std0_k0_reduces_to_is_regression_byte_for_byte` (parametrised over the
regress / no-regress / tolerance / missing-metric / empty cases): the new path
returns the same `bool` object as the legacy path for every input. The full
existing `eval`/`metrics` test suite stays green, which is the byte-compat proof.

## Public API added
```python
# crawfish.metrics
def noise_band(std: dict[str, float], *, alpha: float = 0.05) -> dict[str, float]: ...
def is_regression_variance_aware(
    baseline: dict[str, float], candidate: dict[str, float], *,
    std: dict[str, float] | None = None, alpha: float = 0.05, tolerance: float = 0.0,
) -> bool: ...

# crawfish.eval
class VerifierNotGated(Exception): ...        # raised by the fail-closed precision gate
@dataclass(frozen=True)
class MetricVerdict: ...                       # per-metric CI + adopt/guardrail status
@dataclass(frozen=True)
class GateDecision: ...                        # promote-or-not + per-metric detail

def paired_gate(
    baseline_scores: Mapping[str, Sequence[float]],
    candidate_scores: Mapping[str, Sequence[float]], *,
    primary: str, alpha: float = 0.05,
    guardrails: Mapping[str, float] | None = None,
    n_resamples: int = 2000, seed: int = 0,
) -> GateDecision: ...

def precision_gate(
    decisions: Sequence[bool], labels: Sequence[bool], *,
    min_precision: float, baseline_exists: bool,
) -> float: ...                                # returns measured precision; raises on fail-closed
```
All gates are deterministic: pure arithmetic over recorded scores; the bootstrap
is explicitly seeded.

## Tests
`packages/crawfish/tests/test_gate_algebra.py` — 21 tests covering all 5 acceptance
criteria: (1) `std=0,k=0` byte-for-byte parity with `is_regression`; (2) paired
noise-band rejection (CI straddles 0); (3) rich-rubric Holm does not inflate
false-promotion past `α`; (4) `precision_gate` raises `VerifierNotGated` with no
baseline; (5) clearly-better candidate (CI strictly > 0) promoted. Plus
primary+guardrail design, determinism, and input-validation edges.
