# CRA-213 — AL-T3/TS-6: Objective, cost-regularized loss

The Tuner's acceptance was a pure-quality rule that would promote a 1%-better, 5×-pricier
candidate. It now accepts an optional cost-regularized `Objective` that **re-ranks among
candidates that already pass the hard regression gate** — so cost can never promote a
quality regression.

## Public API added (`crawfish.tuner`)

- `Objective(BaseModel, frozen)`
  - `value(scores, *, cost_usd, ece=0.0) -> float` = `Σ wᵢ·scoreᵢ − λ·cost − μ·ece`.
    Pure arithmetic over **passed-in values**; same inputs ⇒ same scalar.
  - `score(scores, *, cost_usd, ece=0.0) -> ObjectiveScore` — the decomposed result.
  - `quality(scores) -> float` — `Σ wᵢ·scoreᵢ` (absent weight defaults to 1.0; a bare
    objective sums every metric).
  - Fields: `weights: dict[str, float]`, `cost_weight (λ)`, `ece_weight (μ)`,
    `form: ObjectiveForm`, `quality_floor` (ε-constraint), `cost_baseline_usd` (normalizer).
- `ObjectiveForm((str, Enum))` — `LINEAR` (weighted scalarization) / `EPSILON` (minimize
  cost s.t. `quality >= quality_floor`).
- `ObjectiveScore(BaseModel, frozen)` — `value`, `quality`, `cost_penalty`, `ece_penalty`,
  `feasible`.

**`Tuner` gains** (all optional; defaults preserve legacy behavior):
- `objective: Objective | None = None` — when set, acceptance maximizes `Objective.value`
  among regression-clean candidates.
- `pareto: bool = False` — additionally require non-domination on `(quality, cost)` vs the
  running best; `TuneResult.pareto_front` reports the regression-clean non-dominated indices.
- `objective_items: int = 1` — item count handed to `estimate_cost` per candidate.
- `TrialResult.cost_usd` / `TrialResult.objective_value` — recorded per candidate (both
  `None` when no objective is configured; legacy log shape preserved).
- `TuneResult.pareto_front: list[int]` — populated only when `pareto=True`.

## Behavior / acceptance

- `cost_weight=0` ⇒ same winner as today (back-compat; cost term vanishes).
- Two equal-quality candidates ⇒ the cheaper is promoted.
- A 2%-better / much-pricier candidate is rejected for a suitable λ.
- A candidate maximizing the objective but regressing past `-tolerance` is **still** rejected
  by the hard `is_regression` gate (non-negotiable).
- ε-constraint form picks the cheapest feasible candidate above the quality floor; infeasible
  candidates sort below every feasible one.
- `pareto=True` never promotes a dominated candidate.
- Same inputs ⇒ same scalar; per-candidate `cost_usd` from the deterministic `estimate_cost`.

## Cost normalization (ML minor fix)

The cost term divides by `cost_baseline_usd` when set (use the cheapest candidate's cost), so
λ is unit-free and portable. With no baseline the raw dollar cost is used.

## Decoupling

`ece` is a **passed-in value** (AL-T4's CI-bounded calibration metric, ships as 0 until
`calibrate` lands). This module never imports `calibrate`. Per-candidate cost is sourced from
the single cost owner `crawfish.cost.estimate_cost` (no re-implementation of the cost model).
