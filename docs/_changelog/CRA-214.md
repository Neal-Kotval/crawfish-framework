# CRA-214 — AL-T6: the explore-rate dial (`ServingLoop`)

**Milestone 3 (Tunable ML library).**

`LearningLoop.improve` is a one-shot **offline** optimizer — nothing routed a bounded
fraction of **live** items to a trial candidate and fed the outcomes back. The `ServingLoop`
is that serving-time explore/exploit overlay.

## What shipped (`packages/crawfish/src/crawfish/learning.py`)

- **`ServingLoop(promoted, trial, schedule, *, seed, sample_size, min_lift, org_id)`**
  - `route(item_id, ctx) -> ServingDecision` — routes `(1-ε)` of items to the promoted
    best, `ε` to the trial candidate, choosing which items explore by a **seeded hash of the
    recorded `item_id`** (`_explore_hash`: sorted-JSON → sha256, mirroring the cassette-key
    law in `runtime/replay.py`). A replay re-explores **exactly** the same items.
  - `explored_items(item_ids, ctx) -> list[str]` — the deterministic explored subset for a
    batch (side-effect-free query).
  - `graduate(trial_rewards, baseline_rewards) -> GraduationVerdict` — the no-peeking
    graduation gate.
- **`ExploreSchedule`** — the ε dial: `epsilon`, `decay` (decaying-ε:
  `epsilon / (1 + decay·served)`), and a typed `strategy` hook.
- **`ExploreStrategy`** (`(str, Enum)`) — `HASH` (the shipped deterministic router) plus
  reserved `UCB1` / `THOMPSON` hooks (they need only per-arm reward mean + count, already in
  the emission ledger; declared so a future strategy plugs in without an API change).
- **`ServingDecision`** — per-item routing verdict (`item_id`, `explore`, `version`).
- **`GraduationVerdict`** — `decided` / `graduate` / counts / means / reason.

## Acceptance pinned (`packages/crawfish/tests/test_serving_loop.py`)

- `epsilon=0` ⇒ **no exploration** (no-op overlay; every item routes to the promoted best).
- Same `(seed, item_ids)` ⇒ **identical explored subset**; a different seed re-rolls a
  different (still deterministic) subset.
- ε **stops when the budget is exhausted** (`CostBudget.remaining_usd == 0` ⇒ no explore).
- Decaying-ε shrinks the rate as items accumulate (`decay=0` is flat fixed-ε).
- **No peeking:** `graduate` returns `decided=False` until `n_outcomes >= sample_size`
  (pre-registered N) — controls Type-I error under continuous peeking.
- A trial **losing to baseline never graduates** (the promoted best is unchanged); a tie
  does not graduate (strict improvement required); `min_lift` is honoured.

## Design notes (ML blockers from the issue, resolved)

1. **Bare fixed-ε is the weakest bandit** ⇒ a decaying-ε schedule is the default, with a
   typed `ExploreStrategy` hook reserving UCB1/Thompson.
2. **Continuous-peeking / optional-stopping inflates false promotions** ⇒ graduation uses a
   **pre-registered per-trial sample size**: no verdict before N outcomes. (An anytime-valid
   confidence-sequence is the alternative the issue allows; pre-registered N is what shipped
   and is the simpler, equally-valid Type-I control.)
3. ε is bounded by the shared `CostBudget` (autonomy ceiling).

## Security spine

Every stochastic choice derives from one recorded `seed` + the recorded `item_id` (a
trusted, static signal — **never** a fluid value) ⇒ identical explored subset under replay
⇒ identical graduation. The trial graduates only through the eval gate; promotion stays with
`LearningLoop` (eval-gated + reversible via `craw learn --rollback`). Only STATIC knobs are
ever promoted. Both arms are frozen, eval-mode Definitions.
