# CRA-218 — TS-8: constrained / grammar-guided decoding as a runtime-call property

Decode-time constraint is *strictly stronger* than the post-hoc validate+repair loop
(`run.py`): instead of detecting a malformed output and paying a metered `_repair`
re-prompt to fix it, the runtime is told the output **shape** up front, so a malformed
value becomes an *impossible* state. Constrained decoding is a per-call property of the
leaf — it strengthens determinism (moves a parse failure from a retried call to a
non-state), it adds no new control flow.

## Public API added (`crawfish.grammar`)

- `GrammarKind(str, Enum)` — `ENUM` / `REGEX` / `JSON_SCHEMA` (ADR 0004 enum shape).
- `Grammar(BaseModel, frozen)` — a static, declarative constraint on one decoded field.
  Build via the classmethods, not the raw initializer:
  - `Grammar.enum(choices) -> Grammar`
  - `Grammar.regex(pattern) -> Grammar` (compiles the pattern now; fails fast on a bad one)
  - `Grammar.json_object(keys) -> Grammar`
  - `Grammar.from_output_schema(outputs: list[Parameter]) -> Grammar | None` — derive a
    grammar from a Definition's **declared, trusted** output schema; `None` when there is
    nothing to constrain (empty schema, or a single free-text `str` output), so the caller
    leaves `grammar=None` and the run degrades to the ordinary validate path.
  - `to_request_grammar() -> str` — serialize to the per-call `RunRequest.grammar` dialect
    string (the value that rides on the *request*, never the Definition).
  - `satisfies(text) -> bool` — does `text` already meet the constraint.
  - `enforce(text) -> str` — **pure, deterministic** projection of arbitrary text onto the
    constraint surface (enum → snap to a declared member; regex → first match;
    json_schema → recover the first balanced `{...}` object, keep declared keys, canonical
    serialization). Raises `GrammarError` only when no candidate exists at all — never a
    silent coercion.
- `GrammarError(ValueError)` — text that cannot be projected onto the constraint at all.
- `parse(serialized) -> Grammar` — the inverse of `to_request_grammar`; a runtime reads the
  request's grammar string back into a typed constraint, then applies `enforce`.

## Plumbing (`crawfish.run`)

- `Run(..., grammar: Grammar | None = None, decode_seed: int | None = None)`. When `grammar`
  is set the run takes a constrained single-agent path (`_run_constrained`): the grammar is
  serialized onto the per-call `RunRequest.grammar` field, `decode_seed` rides with it, the
  runtime is invoked, and the grammar's `enforce` snaps the result onto the constraint
  surface so validation passes with **no repair**. `grammar=None` opts out (the
  "Let Me Speak Freely?" reasoning caveat) even when the schema could yield one, and the
  ordinary `run_team` validate path is unchanged.
- `Run.repair_count` — observability counter; a grammar-honouring constrained decode keeps
  it at **0** (the acceptance criterion); the unconstrained REPAIR path increments it.

## Runtime contract (`crawfish.runtime.base`)

The per-call decode fields `RunRequest.grammar: str | None` and
`RunRequest.decode_seed: int | None` (added in F-5/CRA-198) carry the constraint and seed.
They are simple typed fields — the eager `RunRequest.model_rebuild()` at import is
unaffected, and `import crawfish` still succeeds.

## Security spine

- **The grammar is static / trusted.** It is built from a Definition's declared output
  schema or from author constants — it has **no constructor that reads a fluid value**, so a
  fluid (untrusted) input cannot set the constraint. The prompt-injection boundary holds:
  the constraint is config, not session data.
- **Run identity.** `grammar` is a per-call property kept **out of** the Definition content
  hash (ADR 0017 / F-5): it constrains the decode surface, it does not version the agent
  (`grammar` / `decode_seed` are absent from `content_dict()` / `version.sha`). The
  output-affecting `decode_seed` enters run identity via the F-1 replay cassette `_key`
  (already folded in by `runtime/replay.py` via `getattr`), so two different decode settings
  do not collide on one cassette. The demo `definition.lock` is unaffected — no grammar is
  folded into any hash.

## Determinism

`enforce` is a pure function of `(text, grammar)`: same text + same grammar ⇒ same
constrained value. With `decode_seed` recorded into run identity, a constrained decode is
bit-for-bit reproducible where the backend honours the seed (per the F-5 `DeterminismTier`).
Tests use `MockRuntime` / pure functions — no live model calls.

## Acceptance (tests in `test_constrained_decode.py`)

- A grammar-honouring run produces a structured field with `repair_count == 0`; the same
  backend with no grammar emits prose → NOT_JSON → a metered repair (`repair_count > 0`).
- Constrained vs unconstrained decode differ predictably (enum snaps to a declared member;
  unconstrained keeps the raw prose).
- Same `decode_seed` ⇒ same recorded constrained result.
- A schema-bearing step opts out with `grammar=None` and takes the ordinary path.
- `grammar` / `decode_seed` do not perturb the Definition `content_sha()` and are absent
  from `content_dict()`; the dialect string round-trips losslessly through `parse`.
