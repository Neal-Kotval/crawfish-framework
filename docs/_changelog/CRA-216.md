# CRA-216 ŌĆö TS-4: abstention as a typed Output discipline

**Milestone 4 (Taming stochasticity).**

Selective prediction ŌĆö *decline rather than hallucinate* ŌĆö is the formal frame for a
reliable agent, but the tameness layer could only **escalate** (`runtime/escalate.py`:
re-run on a stronger model); it could never **give up**. TS-4 adds the missing primitive:
a first-class "I decline to answer" that is a typed Output **value**, not an exception or
a magic string.

## What shipped (`packages/crawfish/src/crawfish/abstain.py`)

- **`Abstention`** ŌĆö a frozen Pydantic payload: `reason`, the *measured* `confidence`
  (`None` when none was reported), the `threshold` it fell under, the producing run's
  `tainted` bit, and the `field` it measured. `as_value()` serialises it to a JSON dict
  tagged with `ABSTENTION_MARKER` (`"_abstention"`), and `from_value()` is the inverse /
  typed guard. The marker travels **in the value** because no Python type survives a
  persisted/replayed Output ŌĆö that's what keeps it routable.
- **`is_abstention(value) -> bool`** ŌĆö a pure, total predicate over any JSON value, safe
  to hand to `Classifier.from_predicates` so a `Router` can branch `Abstention ŌåÆ review`.
- **`abstain_below(threshold, *, field="confidence", reason=None)`** ŌĆö the discipline
  (mirrors `runtime.escalate.confidence_below`, but acts on a frozen `Output` and
  *declines* instead of escalating). It **measures** confidence via
  `escalate.extract_confidence` (a fluid self-report is data, never an instruction), and:
  - confidence `>= threshold` ŌåÆ returns the input Output **unchanged** (act);
  - confidence `< threshold` **or absent** ŌåÆ returns a fresh Output (via `Output.derive`,
    so **taint + lineage propagate**) carrying the `Abstention`. A *missing* confidence
    abstains ŌĆö declining is the fail-safe action and is always allowed.
  - Idempotent: an already-abstaining Output is returned untouched (no double-wrap).
- **`abstain_below_calibrated(report, ...)`** ŌĆö `abstain_below` wired to
  `CalibrationReport.abstention_threshold` (the confidence where observed accuracy crosses
  target, read off `cw.calibrate`'s reliability curve via `escalate.abstention_threshold`)
  ŌĆö the sound default. A raw constant is unsound (the issue's stated risk).

Re-exported from `crawfish.escalate` (`abstain_below`, `abstain_below_calibrated`,
`is_abstention`, `Abstention`) so the confidence vocabulary reads as one surface;
implementation lives in `abstain.py` (a bottom-of-file import avoids the
`abstain ŌåÆ escalate` cycle).

## Acceptance pinned (`packages/crawfish/tests/test_abstain.py`)

- A **0.5-confidence** result under `abstain_below(0.7)` yields an `Abstention`; a 0.95
  result passes through unchanged; the threshold boundary is inclusive (`>=` acts).
- `Abstention` type-checks as a valid `Output` value and is **routable** ŌĆö a `Router` with
  an `is_abstention` predicate branches it to `review`, confident outputs to `proceed`.
- A **calibration-derived** threshold differs from a naive constant on a **mis-calibrated**
  fixture: a 0.6-confidence answer that `abstain_below(0.5)` would act on is *declined* by
  `abstain_below_calibrated` (derived ~0.95).
- `calibrate`'s `abstention_rate` equals the share `abstain_below(report.abstention_threshold)`
  declines over the same outputs.
- An abstaining run **still charges what it spent**: the discipline is pure over the
  produced Output and touches no budget, so the cost of the run that produced the
  low-confidence answer stands (abstention is post-hoc, never a refund).
- **Taint propagates** into the `Abstention` (and lineage threads through `derive`).

## Security spine

The confidence is **measured data** (`extract_confidence`), never trusted as an
instruction. Abstaining is **fail-safe** ŌĆö a missing/unparseable confidence declines, and
declining is always allowed. Taint propagates into the `Abstention` so an abstention
derived from fluid input stays tainted (it can never become a Sink target or idempotency
key). Deterministic: a pure threshold over a recorded number; `Abstention` is frozen.
