# CRA-215 — TS-1: typed quorum / self-consistency aggregator (`QuorumRuntime`)

**Milestone 4 (Taming stochasticity).**

Self-consistency (sample N, take consensus) is the cheapest, best-attested variance reducer
and the purest expression of the thesis — **N stochastic leaves reduced by a deterministic
vote**. The aggregator layer had only `collect/concat/count/dedupe`, and `fan_in` reduces
*distinct* items, not *k samples of one item*. `QuorumRuntime` is that missing operator.

## What shipped (`packages/crawfish/src/crawfish/runtime/quorum.py`)

- **`QuorumRuntime(inner, *, k, consensus, default_text, base_seed, early_stop, alpha,
  min_k)`** — wraps any inner `AgentRuntime`, samples the *same* `RunRequest` `k` times
  (each a distinct seeded leaf charging the shared budget through `inner`, the `escalate.py`
  pattern), and reduces the `k` results by a typed, pure consensus vote.
  - `run(request, ctx) -> RunResult` — the plain `AgentRuntime` seam (winner text only).
  - `run_quorum(request, ctx) -> QuorumResult` — the richer shape: winner `RunResult`,
    **aggregate taint**, the `ConsensusResult` (tally + runner-up gap), and the samples.
    `RunResult` has no taint field (taint lives on `Output`), so the aggregate taint is
    surfaced here for callers wrapping the winner into an `Output`.
  - `k` defaults to the Definition's tunable **`sample_k`** knob (AL-T1, via
    `request.resolved_decode()`) so the Tuner can search the cheapest `k` hitting a
    reliability target; an explicit `k` overrides; floor `min_k=3`.
- **`ConsensusFn`** (ABC) — `key_of(result)` (the vote equivalence class) + `consensus(samples)`,
  on one object so the running-leader early-stop and the final tally share one notion of
  equality.
- **`MajorityVote` / `majority_vote(*, field, max_cardinality_ratio)`** — modal-output
  estimand. **Mandatory canonicalization** collapses semantically-equal outputs
  (`{"a":1,"b":2}` and `{"b":2,"a":1}` → one candidate); `field` votes over a dotted
  sub-field. Ties break to the first-seen key.
- **`quorum_output(result, *, produced_by, tainted, output_schema)`** — wrap a quorum
  result into a typed, correctly-tainted `Output`.
- **`QuorumResult` / `ConsensusResult` / `Sample`** — the typed value shapes.
- **`QuorumAbstention`** — raised when the vote is ill-defined and no default is declared.

## Acceptance pinned (`packages/crawfish/tests/test_quorum.py`, 14 tests)

- k=5 over `RecordReplayRuntime` returns the majority typed Output **bit-for-bit**; **k
  distinct cassette keys** (via the F-1 `sample_index` coordinate) — no collision into one
  unanimous cassette; replay makes **zero** inner calls.
- Budget below k× per-call **stops early** and **never exceeds the ceiling** (`charge` never
  raises `BudgetExceeded`); each sample charges the shared budget.
- Sequential proportion test (F-8) returns once the lead is statistically real (Wilson lower
  bound on the leader's share > 0.5); same base seed ⇒ identical sample count + winner.
- `majority_vote(field="label")` collapses reordered records; a high-cardinality
  (all-distinct) vote **abstains** to the declared default; no default ⇒ `QuorumAbstention`.
- Cancel between samples honoured.
- Winner tainted **iff any** sample tainted (taint union).
- `k` defaults from the Definition's `sample_k` knob.

## Design notes (blockers from the issue, resolved)

1. **Cassette collision (SECURITY blocker):** k samples under one unsalted key collide into
   one cassette (unanimous no-op, variance 0). **Resolved** via the F-1
   `ExecutionCoordinate(sample_index=i)` — F-1 has landed, so `QuorumRuntime` passes the
   coordinate to a `RecordReplayRuntime` (k distinct cassettes) rather than refusing to run.
2. **Estimand + ill-defined plurality:** the estimand is the **modal output**
   (`majority_vote`), with mandatory canonicalization; on high-cardinality outputs where
   plurality is ill-defined, it **abstains** (TS-4).
3. **Optional-stopping (`early_stop`):** the fixed `0.8` had no statistical basis. **Replaced**
   with a **sequential proportion test** (stop when a Wilson lower bound on the leader's lead
   exceeds 0.5, or after the pre-registered k) — F-8.
4. **Aggregate taint (SECURITY):** the consensus winner is tainted if **any** sample was
   tainted (union) — a vote does not launder taint (ALG-7 conformance).

## Security spine

Each sample is an isolated leaf with a distinct, derived `decode_seed` (replayable). The
consensus reduction is **pure** over the recorded `RunResult.text` — no model call, no I/O,
no wall-clock. The run is bounded by `k` + the cost budget + the cancel token. Fluid sample
outputs are **data**, never instructions; the vote tally and the **declared** default are
static/trusted, never fluid-derived. Ties / no-majority resolve to the declared default
(Router dead-letter parity) — never a silent pick. Taint is the **union** across the `k`
samples.

## Deferred (dependencies not yet landed)

- **`rubric_argmax`** (argmax of expected rubric, winner's-curse abstention via the
  `cw.calibrate` noise band, tie-break through a gated **Verifier**, CL-2) is specced but
  deferred to its dependencies (AL-T4 noise band wiring + CL-2). The `ConsensusFn` ABC is the
  extension point: `rubric_argmax` is a second `ConsensusFn` that plugs in without an API
  change. `MajorityVote` (the modal-output estimand) is the shipped consensus.
